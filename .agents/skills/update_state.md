# Skill: Updating PROJECT_STATE.md

**Trigger**: After completing ANY coding task or milestone.

## Rules

1. **Mandatory Step**: You MUST update `PROJECT_STATE.md` before concluding your task. This is non-negotiable.
2. **File Completion**: 
   - Locate the file you just worked on in the `FILE COMPLETION STATUS` section.
   - Change `[ ]` to `[DONE - YYYY-MM-DD]`.
   - Add a brief one-line note describing its purpose next to it.
3. **API Routes Tracker**:
   - If you implemented an API route, find it under `API ROUTES TRACKER` and mark it `[DONE]`.
4. **Change Log**:
   - Append a new entry at the absolute bottom under `## CHANGE LOG`.
   - Format: `[YYYY-MM-DD] [file or folder] — what was done`.
5. **Decisions Log**:
   - If you made a structural or architectural decision not explicitly dictated by `AGENTS.md`, record it under `## DECISIONS LOG`.

**Warning**: Failing to update this file breaks project tracking for the entire agent swarm.
